## 三、执行中与预期不符之处

### 3.1 最大意外：16 条风险中 15 条已在前序 Sprint 落地

**预期：** 每一条 Pending Review 都需要按方案文件执行文档修改。
**实际：** 仅 ARC-09-P1-001 和 12_规范 Appendix B 的 SSOT 注释需要新增修改。其余 15 条已在先前 Sprint（16/19/20/21/22）中由并行 Agents 自动执行了内容落地，但跟踪表状态未同步更新为 Resolved。

**原因分析：** 治理线的 Sprint 16~22 专注于"方案设计和批准"，批准的方案由多路并行 Agent 在架构文档中直接实施，但缺乏将实施结果同步回跟踪表的闭环步骤。这是治理流程中的一个自动化缺口。

### 3.2 编辑级联问题

使用 Edit 工具批量更新 18_跟踪表时，因旧字符串匹配范围过大，发生数次级联覆盖（ARC-16-P1-002 获得 ARC-16-P1-001 的文本、ARC-16-P1-008 表结构损坏）。已逐一修复。建议未来处理大块表格更新时优先使用逐行替换而非大块匹配。
