### 3. 方案本身的逻辑漏洞或遗漏

**ARC-15-P1-006 方案要求 longitude 为必填，但 13_API 两处请求示例不一致**

- 方案明确要求 `longitude` 和 `timezone` 设为必填，**但没有处理 `latitude`**。当前 API 设计中 `latitude` 是可选字段（它本身没有时柱计算相关的技术必要性）。
- 已按"longitude 必填 + 示例值 116.4（北京）、latitude 可选 + 示例值 39.9"执行。这个处理方式应该是合理的。

**ARC-15-P1-006 方案的字段与 13_API 现有字段不完全匹配**

- 方案定义 `BaZiRequest` 有 `birth_year`/`birth_month`/`birth_day`/`birth_hour`/`birth_minute` 但无 `birth_place`/`name`/`is_primary`
- 13_API 现有的请求中有 `birth_place`（string, optional）、`name`（string, optional）、`is_primary`（bool, optional）
- 这些 UI 相关字段不应被删除，它们和管理排盘列表相关。本次修改保留了这些字段，仅新增了 `longitude`/`latitude`。
