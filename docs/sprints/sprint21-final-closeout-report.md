## 2. Part B 方案核心设计思路（一句话总结）

> **普通文档升阶由产品负责人按"基本就绪标准"（4 项，比 ADR 的 5 项更轻）单点确认，不绑定 ADR 的强制时限、不设超时自动升阶——把治理成本压到与文档影响面匹配的最低水平。**

关键差异化设计：
- **升阶条件**取 4 项（内容完整 / 一致性 / 定位清晰 / 元信息规范），比 ADR 少"影响分析"和"版本兼容性"。
- **权限**：唯一升阶权归产品负责人；AI 只能建 Draft、提请评审，不得自标 Approved。
- **不设超时自动升阶**（与 ADR 不同）——规避"无人审批即默认批准"，宁可长期停 Draft。
- 方案文件状态 🟡 Pending Review，**未落地 12_规范**，等产品负责人评审后再决定是否写入 §5.2.1。
