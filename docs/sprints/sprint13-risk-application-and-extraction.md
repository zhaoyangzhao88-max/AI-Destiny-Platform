#### ARC-15-P2-003 — WS 心跳 pong 超时无缓冲

**来源：** `docs/15_Phase2架构审计报告.md` §7.3 P2#3

**原始审计发现（15_报告 §7.3）：**

> | 3 | WS 心跳 pong 超时无缓冲 | `14_WS` 7.3 | pong 超时设为 ping 间隔 2 倍 (60s)

**风险跟踪表完整记录（18_跟踪表 §4.3）：**

| 字段 | 内容 |
|------|------|
| **问题描述** | 14_WS.md 7.3 节 pong timeout = ping interval（30s），无缓冲余量，网络抖动可导致误判断连 |
| **等级** | 🟢 P2 |
| **影响** | 网络不稳定时可能频繁误断连 |
| **建议措施** | pong 超时设为 ping 间隔 2 倍（60s） |
| **状态** | 🟢 Open |
| **负责人角色** | Architect Agent |
| **关闭条件** | 14_WS.md 7.3 节 pong timeout 更新为 60s |

**关键上下文：**
- 涉及文件：`docs/14_WebSocket实时通信协议设计.md` §7.3 心跳机制
- 当前设定：30s ping / 30s pong timeout / 90s idle timeout
- 问题：pong timeout 等于 ping interval，网络延迟抖动即可触发断连
- 建议：pong timeout = 60s（2 × ping interval），与业界常见做法一致
