## 7.1 流程总览

八字分析采用 **6 步动态流程**（ADR-009）：

```
┌─────────────────────────────────────────────────────┐
│                   八字分析流程                       │
│                                                     │
│  出生信息 ──→ 排盘 ──→ AI初步分析 ──→ AI动态追问    │
│                                                    │
│   AI动态追问 ──→ 用户反馈验证 ──→ 最终报告          │
│       │                              │              │
│       └── 若需补充 ──→ 回到AI分析 ──┘              │
└─────────────────────────────────────────────────────┘
```

**关键特征：**
- 不是一次性输出，而是多轮动态交互
- 用户反馈可触发分析回路
- 分析深度随交互轮次递增

## 7.2 第一步：出生信息采集

| 字段 | 必填 | 说明 |
|------|------|------|
| 出生年 | ✅ | 公历年份 |
| 出生月 | ✅ | 公历月份 |
| 出生日 | ✅ | 公历日期 |
| 出生时辰 | ❌ | 允许未知（ADR-011），未知时用子时暂代并标注 |
| 性别 | ✅ | 排大运需要 |
| 出生地点 | ❌ | 用于真太阳时校准 |

**采集方式：** AI 引导对话 + 快速填写表单混合（ADR-008）

## 7.3 第二步：排盘

排盘是将出生时间转换为四柱八字的结构化过程。

**输入：** 出生年、月、日、时 + 性别 + 地点  
**输出：** 四柱八字盘面数据

排盘结果包含：

| 维度 | 内容 |
|------|------|
| **四柱** | 年柱、月柱、日柱、时柱（天干+地支） |
| **五行** | 四柱五行的分布与旺衰 |
| **十神** | 日主与其他天干的关系 |
| **大运** | 起运年龄、大运十步排列 |
| **当前流年** | 当前年份的干支与运势影响 |

> 💡 排盘引擎选型 ——【待确认】  
> 可参考 `bazi-skill-dist` 的排盘方法论或 `bazi-master-analysis` 的 Python 排盘脚本。

## 7.4 第三步：AI 初步分析

基于排盘结果，AI 进行第一轮分析：

**分析维度：**
1. **日主旺衰** —— 日主强弱、是否需要调候
2. **格局判定** —— 正格/变格/从格等
3. **用神忌神** —— 喜用五行、忌讳五行
4. **十神配置** —— 印比食财官杀的分布与关系
5. **大运趋势** —— 当前所处大运，未来大运走向

**输出格式：** 结构化的初步分析报告，标注"推测"层级（ADR-012）

## 7.5 第四步：AI 动态追问

初步分析后，AI 在发现以下情况时会主动追问：

| 触发条件 | 追问方向 |
|----------|----------|
| 日主过旺/过弱 | "你是否感觉容易走极端？" |
| 某十神异常突出 | "你的事业是不是和 X 领域相关？" |
| 大运有重大转折 | "最近几年是不是有重大变化？" |
| 时辰未知 | "有没有什么关键的人生时间点？" |

**追问原则：**
- 一次只问 1-2 个问题，避免信息轰炸
- 问题基于排盘结果，有命理依据
- 用户回答后更新分析

## 7.6 第五步：用户反馈验证

用户对 AI 的分析结果进行反馈：

| 用户行为 | 系统处理 |
|----------|----------|
| 确认"是的，很准" | 将对应推测标记为"事实"级别（ADR-012） |
| 否认"不对，完全不是" | 保留推测记录，降低该分析维度权重 |
| 补充"部分对，但……" | 结合补充信息重新分析 |
| 不置可否 | 保持"推测"层级，等待后续验证 |

**核心原则（ADR-012）：** 用户确认的事实优先于任何 AI 推测。

## 7.7 第六步：最终报告

经过动态追问和用户验证后，生成最终分析报告。

**报告结构：**

```
1. 📋 命盘总览（四柱 + 五行 + 十神一览）
2. 💪 旺衰与用神（日主强弱、喜用忌神）
3. 🏛️ 格局分析（格局类型、层次判断）
4. 👥 十神解析（六亲、事业、财运、健康等维度）
5. 🛤️ 大运流年（当前大运、未来趋势）
6. ✨ 辅助参考（神煞、命宫等）
7. 📝 人生建议（基于分析的战略性建议）
8. ✅ 可信度标注（事实/推测/待验证）
```

> ⚠️ 报告模板与详细结构 ——【待确认】

## 7.8 多轮迭代机制

分析流程支持多轮迭代：

```
第一轮分析 → 用户反馈 → 深度分析 → 用户再反馈 → …… → 最终报告
```

**迭代终止条件：**

| 条件 | 说明 |
|------|------|
| 用户明确满意 | 用户说"可以了""很好"等 |
| 分析维度已覆盖 | 所有主要分析维度均已输出 |
| 追问无新发现 | 用户反馈不再产生新的分析信息 |
| 达到最大轮次 | 预设最大迭代次数（具体次数待确认） |

## 7.9 分析完成条件

一份完整的八字分析满足以下条件时标记为"完成"：

- [ ] 四柱排盘完成（未知时辰标注清楚）
- [ ] 旺衰与用神分析完成
- [ ] 格局判定完成
- [ ] 十神分析完成
- [ ] 大运流年分析完成
- [ ] AI 追问已完成
- [ ] 用户反馈已记录
- [ ] 数据层级标注完成
