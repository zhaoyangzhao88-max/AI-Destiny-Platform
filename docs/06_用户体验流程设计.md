## 6.1 用户旅程总览

```
发现 → 首次对话 → 信息采集 → 八字分析 → 日常陪伴 → 深度探索
                                              ↕
                                        人生档案累积
```

## 6.2 首次使用流程

```
APP 启动
    │
    ▼
首次欢迎语（融合型风格，ADR-010）
    │
    ▼
是否知道出生时间？
    ├── 知道 → 填写出生信息
    └── 不知道 → 跳过 / 后续逐步补充
    │
    ▼
AI 引导 + 快速填写混合模式采集信息（ADR-008）
    │
    ▼
八字排盘 & AI 分析
    │
    ▼
用户反馈验证
    │
    ▼
进入首页（知命陪伴 + 档案进度，ADR-007）
```

## 6.4 首次信息采集流程

**模式：** AI 引导 + 快速填写混合模式（ADR-008）

**AI 引导模式：**
```
知命：请问你的出生日期是哪一天？
用户：1990年8月15日
知命：知道出生时辰吗？比如几点几分？
用户：不知道
知命：没关系。我可以在后续聊天中慢慢推测。先告诉我你的性别？
...
```

**快速填写模式：**
提供一个结构化表单，包含：
- 出生年 / 月 / 日 / 时（时分可选）
- 性别
- 出生地点（用于真太阳时校准）
- 必填项：出生日期 + 性别
- 选填项：出生时辰、出生地点

## 6.5 出生时间处理策略

**核心规则（ADR-011）：**

| 情况 | 处理方式 |
|------|----------|
| 用户知道完整出生时间（年月日时分） | 正常排盘，使用标准流程 |
| 用户不知道时辰 | 标记"时辰未知"，排盘时省略时柱，AI 分析中注明 |
| 用户只记得大概时间段 | 使用时间段推断，标记为"推测层级" |
| 用户后续提供补充信息 | 通过版本历史机制（ADR-013）更新档案 |
| 从人生事件反推 | AI 根据用户提供的人生事件逆向推算，结果标记为"待验证"（ADR-012） |

## 6.6 日活用户旅程

```
每日打开 APP
    │
    ▼
首页 → 知命打招呼（根据时间/节气/用户状态自适应）
    │
    ├──→ 查看档案进度 → 补充新的人生事件
    │
    ├──→ 与知命聊天 → 日常问答 / 命理探讨 / 生活分享
    │
    ├──→ 查看分析报告 → 回顾之前的八字分析 / 大运推演
    │
    └──→ 探索新功能 → 深度分析 / 流年提醒 / 关系合盘（待定）
```
