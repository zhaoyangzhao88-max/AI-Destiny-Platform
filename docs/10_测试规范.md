## 10.2 测试挑战

本项目测试面临以下特殊挑战：

| 挑战 | 说明 |
|------|------|
| **心理测试不可自动化** | 命理分析"准不准"本质上是主观体验，难以用单元测试覆盖 |
| **AI 输出不确定性** | LLM 生成的命理分析每次输出可能不同，难以断言精确结果 |
| **排盘确定性** | 排盘结果是确定性的（同样的输入得到同样的四柱），但 AI 分析部分不确定 |
| **边界情况多** | 未知时辰、闰月、不同时区、真太阳时校正等 |

## 10.3 测试层级初步建议

| 层级 | 覆盖内容 | 建议策略 |
|------|----------|----------|
| **单元测试** | 排盘引擎、数据转换、工具函数 | 确定性输入输出，可完全自动化 |
| **集成测试** | API 接口、数据库操作 | 标准 REST API 测试 |
| **AI 输出测试** | 报告结构完整性、术语使用正确性 | 结构校验 + 关键词检测 |
| **用户验收测试** | 完整产品体验 | 手动测试为主 |

## 10.4 命理测试的特殊性

**排盘测试（可自动化）：**
```
输入：1990-08-15 12:00, 男, 北京
预期输出：
  年柱：庚午
  月柱：甲申
  日柱：戊寅
  时柱：戊午
```

**AI 分析测试（半自动化）：**
- 输出格式符合预设模板结构
- 使用的命理术语正确
- 不出现矛盾逻辑（如同时说"日主极旺"和"日主极弱"）
- 不出现禁止表述（如"你一定"、"你肯定"等绝对化表述）
