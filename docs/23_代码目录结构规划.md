## 1. 顶层目录结构

基于 ADR-014~020 已确认的技术栈（PySide6 + Vue 混合架构、FastAPI 核心服务、SQLite 本地优先），顶层目录规划如下：

```
AI-Destiny-Platform/
├── src/
│   ├── core/            # Python 核心服务（FastAPI 应用）
│   ├── desktop/         # PySide6 客户端外壳
│   ├── web/             # Vue 渐进式 UI 模块
│   └── tests/           # 测试套件
├── docs/                # 项目文档（已有）
├── scripts/             # 开发/构建辅助脚本
├── requirements/        # Python 依赖拆分
├── pyproject.toml       # Python 项目配置
└── README.md
```

### 设计原则

1. **按层分目录，不按功能分目录** — 核心服务、桌面外壳、Web UI 分属不同目录，各自独立构建
2. **核心服务与 UI 解耦** — `src/core/` 不依赖 `src/desktop/` 或 `src/web/`，反之亦然
3. **测试目录独立** — `src/tests/` 与源码目录平级，不混入各模块
4. **与 08 号文档架构对齐** — 每个目录映射到 08_系统架构设计.md 的对应模块

### 4.2 渐进引入说明

Vue 模块遵循 ADR-018 的渐进式引入原则：

- **V1 初期**：Vue 模块仅作为 QWebEngineView 内嵌页面，由 PySide6 主窗口托管
- **V1 后期**：复杂交互页面（AI 聊天、高级报告、数据可视化）逐步从 PySide6 原生控件迁移至 Vue
- **V2**：Vue 代码可直接复用于独立 Web 端，无需重写

## 6. Walking Skeleton 迁移路径

### 6.1 现有验证代码清单

`docs/sprint27-walking-skeleton/scripts/` 下已验证的代码：

| 文件 | 内容 | 行数 | 已验证功能 |

### 6.2 迁移映射表

| 现有代码 | 目标路径 | 说明 |
|---------|---------|------|
| `bazi_core.py` 中 `Pillar` / `BaZiResult` 数据类 | `src/core/domain/models/bazi_chart.py` | 领域模型，保持字段一致 |
| `bazi_core.py` 中 `HEAVENLY_STEMS` / `EARTHLY_BRANCHES` / `HIDDEN_STEMS_MAP` / `SIXTY_CYCLE_NAYIN` 常量 | `src/core/infrastructure/bazi/constants.py` | 排盘引擎常量 |
| `bazi_core.py` 中 `calculate_bazi()` 函数 | `src/core/domain/services/paipan.py` | 领域服务——排盘计算 |
| `bazi_core.py` 中 `_ganzhi_to_index()` / `_nayin_from_ganzhi()` 等辅助函数 | `src/core/infrastructure/bazi/utils.py` | 工具函数，按需引入 |
| `bazi_core.py` 中 `self_check()` | `src/core/infrastructure/bazi/engine.py` | 继承 BaziEngine ABC 后作为 self_check 实现 |
| `ai_interpret.py` 中 `interpret_bazi()` | `src/core/infrastructure/ai/llm_client.py` | LLM 调用封装，需适配 LLM Gateway 接口 |
| `run_demo.py` | `src/core/scripts/` 或作为集成测试模板 | 演示脚本，开发期可保留 |

### 6.3 迁移注意事项

1. **保持接口兼容**：迁移后的 `PaipanService` 应实现 `domain/interfaces/bazi_engine.py` 中定义的 `BaziEngine` 抽象接口（08 §8.8.1），确保引擎可替换
2. **保留 self_check**：`self_check()` 是 ARC-15-P1-006 要求的强制校验机制，迁移后需作为 BaziEngine 实现的一部分
3. **不重写已验证逻辑**：已有代码经过 5 种独立方法交叉验证，迁移时仅做目录调整和包导入适配，不修改核心计算逻辑
4. **后续扩展**：正式版还需补充 `YongShen` / `XiShen` / `JiShen` 推导（walking skeleton 中的占位需要在正式代码中实现）
