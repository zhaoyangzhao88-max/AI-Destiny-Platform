## 6. AI 协作规则

### 6.1 必须遵守

1. **所有参与本项目的 AI（包括但不限于 Claude Code、ChatGPT、其他大模型）必须先完整阅读 `docs/` 目录下的核心文档**，包括：
   - `01_项目简介.md`
   - `02_产品定位与愿景.md`
   - `04_ADR_产品决策记录.md`
   - `05_知命_Persona角色设定.md`
   - `HANDOVER.md`

2. **不得直接修改已确认的架构决策（ADR）**。如果发现决策存在问题，必须：
   - 在 `docs/04_ADR_产品决策记录.md` 的附录中提出质疑记录
   - 在 `docs/11_AI协作日志.md` 中记录建议
   - **等待产品负责人确认后才能修改**

3. **每次 AI 执行任务后必须在 `docs/11_AI协作日志.md` 中追加记录**。

4. **标记【待确认】的内容，AI 不得自行假设答案。**
   - 必须呈现为待决策状态
   - 可提供建议方案，但必须注明"建议"而非"决定"

### 6.2 建议遵循

1. 如果需要参考命理知识体系，优先查看 `../bazi-skill-dist/`（四家投票方法论）和 `../bazi-master-analysis/`（滴天髓分析体系）
2. 修改现有文件时，先展示受影响的行范围
3. 不要重写无关代码，保持外科手术式修改风格

### 6.3 禁止行为

- ❌ 禁止在未阅读核心文档的情况下开始工作
- ❌ 禁止未经确认修改已确认架构
- ❌ 禁止自行填补【待确认】内容
- ❌ 禁止删除或覆盖用户数据

你好，我是「知命 AI 人生档案顾问」项目的交接文档。

如果你被唤醒来参与本项目，请按以下步骤启动：

### Step 1：阅读核心文档

```
先读：
  docs/HANDOVER.md          ← 当前文件
  docs/01_项目简介.md        ← 项目背景
  docs/02_产品定位与愿景.md   ← 产品方向

  docs/05_知命_Persona角色设定.md ← AI 角色设定
```

### Step 2：了解当前状态

```
  docs/03_当前开发状态.md     ← 已完成 & 待办
  docs/06_用户体验流程设计.md ← UX 设计
  docs/07_AI动态分析流程.md  ← 分析流程
  docs/08_系统架构设计.md    ← 架构框架
```

### Step 3：检查 AI 协作日志

```
  docs/11_AI协作日志.md      ← 之前 AI 做了什么、遇到了什么问题
```

### Step 4：开始工作

- 如果是新任务：先在 `11_AI协作日志.md` 中创建新条目

- 如需补充文档内容：遵守现有格式约定
- **重要：发现任何问题或建议，如实记录，不要为了执行而执行。**
